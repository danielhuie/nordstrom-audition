Project Overview:

Dataset - City of Seattle Bicycle Racks (https://data.seattle.gov/resource/vncn-umqp.json)

Assumptions - We are collecting data from the City of Seattle Bicycle Racks dataset. Using the dataset, I will be creating a visualization in an informative way that illustrates the numerous racks in the city according to location. The User Interface will allow the user to search for a particular location and filter it in the map.

Questions -
What is the best way to display the data?
How many datasets do we want to compare?
What properties do we want to filter?
How could we make loading the JSON data more efficient since we are dealing with thousands of data points?
Is a heatmap more effective than individual data points?

Idea - Implement Google Maps API to visualize the data and implement a data filter. Create a form to sort data and ask user what they want visualized.

Technologies used - jQuery, Javascript, JSON, HTML5, Bootstrap, CSS3, Google Maps API

Why illustrate individual data points - Individual data points allows us to see where the bike racks are physically located. It also helps us to visualize the density of the racks positioned adjacent to one another, thus allowing policy makers to make quick decisions on where to install future bike racks.

Project Purpose

By displaying bike racks located throughout the city of Seattle, city officials are able to visually locate bike racks. This is particularly useful for urban and environmental planning purposes so city officials can strategically place bike racks across the city without creating too much clutter in one area. In the event that a rack needs maintenance, they are able to filter the results and identify the rack based on condition.

A Work in Progress...

Although I have experience with AngularJS, integration with the Google Maps API is better off using jQuery because DOM manipulation is easier and more organized in the code base. However, future implementations of AngularJS can include creating a table listing the data in the order search results are fetched. Pagination will also be included so that data is loaded from the server and not all at once on the client-side.

I will also build a detailed filtering system that allows the user to toggle between a heat map and the static marker display. This allows the user to visualize the data in two forms to compare and contrast population density with respect to bike rack location.
